# 01 - 项目演示与介绍

---

## 一、项目定位与价值

### 解决什么问题

电商平台每天面对大量重复性客户咨询：商品优惠、退换货流程、物流查询、尺码推荐……传统做法要么纯人工（成本高），要么硬编码规则（体验差）。

本项目是一个 **生产级智能客服系统**，核心思路：

> **LLM 大模型 + 知识库检索 + 规则兜底 = 智能且可靠的客服**

关键特点：
- **双引擎架构**：LLM 可用时走大模型生成回复，LLM 不可用时自动降级为关键词规则引擎，保证系统永远可用
- **知识驱动**：回复基于真实商品、活动、FAQ 数据，不是胡编乱造
- **多轮对话记忆**：同一会话内自动携带最近 10 条历史消息，LLM 能理解上下文连续对话
- **透明可观测**：每次对话都展示完整推理过程（场景识别 → 意图分析 → 知识检索 → 生成回复）
- **真实与模拟分明**：Tab 标签的 tooltip 角标清晰标注每个功能是否接入真实对话链路

### 适用场景

- 电商客服、售前咨询、售后工单
- 任何需要「知识库 + LLM」结合的对话场景

## 二、核心功能演示

### 功能1：对话中心

> 页面路径：`/customer-service/chat`  
> 【截图位置：对话中心四栏布局全貌】

四栏布局设计：

| 区域 | 功能 |
|------|------|
| **左侧 - 常见问题** | QuestionSidebar.vue，预设常见问题，点击直接发送 |
| **中间 - 对话窗口** | ChatInterface.vue，iPhone 风格聊天界面，支持会话创建/切换/删除 |
| **右上 - 知识选择** | KnowledgeSelect.vue，手动选择商品/活动作为对话上下文 |
| **右下 - Agent 工作区** | AgentWorkArea.vue，实时展示推理过程（6步推理链） |

**演示操作**：
1. 点击「新建会话」
2. 输入"最近有什么优惠活动吗？"
3. 观察右下角推理过程：输入解析 → 场景识别(售前咨询) → 意图分析(商品优惠查询) → 情绪判断 → 知识检索 → 生成回复
4. 在右上角选择一个商品，再次提问"这个商品有折扣吗？"，观察知识检索结果变化
5. **多轮对话演示**：继续追问"那尺码怎么选？"，观察 AI 能记住之前选的商品（后台日志可看到传入了历史消息）

### 功能2：知识中心

> 页面路径：`/customer-service/knowledge`  
> 【截图位置：知识中心 Tab 页全貌】

5 类知识数据管理，每类独立 Tab。鼠标悬停在 Tab 标签上会显示 **tooltip 角标**，说明该类型在对话链路中的作用和接入状态：

| Tab | 对话链路接入 | 说明 |
|-----|:----------:|------|
| **🛍️ 商品知识** | ✅ 真实 | 对话时作为商品上下文注入 LLM，支持前端手动选择 |
| **🎁 活动知识** | ✅ 真实 | 对话时作为优惠上下文注入 LLM，支持前端手动选择 |
| **❓ FAQ问答** | ✅ 真实 | 对话时关键词匹配，命中后作为回复参考注入 LLM |
| **📚 行业知识（模拟）** | ❌ 模拟 | CRUD 真实可操作，未接入对话链路，预留 RAG 向量检索扩展 |
| **🎯 解决方案（模拟）** | ❌ 模拟 | CRUD 真实可操作，未接入对话链路，预留流程编排扩展 |

> **教学要点**：5 个 Tab 的 CRUD（新增/编辑/删除）功能全部真实可操作，区别在于前 3 类数据会在对话时被检索并注入 LLM 上下文，后 2 类目前仅做数据管理，留作架构扩展演示。

**演示操作**：
1. 切换各 Tab 查看不同类型知识，**悬停 Tab 标签查看 tooltip 说明**
2. 点击「新增」按钮，弹出表单对话框，填入商品信息并保存（真实写入数据库）
3. 编辑一条现有知识，观察表单回填效果
4. 回到对话中心，询问相关问题，观察该商品是否被检索命中
5. 对比「行业知识（模拟）」Tab，演示 CRUD 可用但不影响对话结果

### 功能3：场景配置

> 页面路径：`/customer-service/scene`  
> 【截图位置：场景配置三个 Tab 页】

三个配置维度，**全部标注为（模拟）**，表示 CRUD 功能真实可操作，但尚未接入对话核心链路。鼠标悬停 Tab 可查看 tooltip 详细说明：

| Tab | 对话链路接入 | 说明 |
|-----|:----------:|------|
| **🎯 意图管理（模拟）** | ❌ 模拟 | CRUD 真实，意图识别当前由 LLM/关键词引擎完成，未从此表读取 |
| **📝 提示词模板（模拟）** | ❌ 模拟 | CRUD 真实，「测试」按钮真实调用 LLM，但对话链路使用代码中硬编码的提示词 |
| **⚙️ 场景规则（模拟）** | ❌ 模拟 | CRUD 真实，启用/禁用真实，未接入对话链路，预留规则引擎扩展 |

> **教学要点**：场景配置模块展示了「管理后台数据建模 + 预留扩展接口」的设计思路。当前对话链路中的意图识别和提示词均在代码中实现，这些配置表预留了动态化的扩展路径（如运行时从数据库加载意图规则/提示词模板）。

**演示操作**：
1. 查看现有意图配置，**悬停 Tab 标签查看 tooltip 说明**
2. 创建一条新的提示词模板，点击「测试」按钮观察 LLM 真实返回效果
3. 创建/启用/禁用一条规则
4. 解释为何标注「模拟」—— CRUD 可用但对话链路暂未消费这些配置

## 四、技术栈说明

### 后端

| 技术 | 版本 | 用途 |
|------|------|------|
| Spring Boot | 3.4.5 | 核心框架 |
| Spring Data JPA | - | ORM 持久化，自动建表 |
| MySQL | 8.0 | 关系型数据库 |
| Druid | 1.2.23 | 数据库连接池 + SQL 监控 |
| Redis + Redisson | - | 缓存 + 分布式锁 |
| Lombok | 1.18 | 代码简化 |
| Java | 17 | 语言版本 |

### 前端

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue | 3.5 | 响应式框架 |
| Vite | 7.x | 构建工具 |
| Element Plus | 2.11 | UI 组件库 |
| Pinia | 3.x | 状态管理 |
| Axios | 1.12 | HTTP 请求 |
| TypeScript | 5.6 | 类型安全 |
| Tailwind CSS | 4.x | 原子化样式 |

### LLM 集成

| 项目 | 说明 |
|------|------|
| 网关 | OneRouter（OpenAI 兼容 API） |
| 模型 | gpt-4o-mini（可切换） |
| 协议 | Chat Completions API |
| 降级 | LLM 不可用时自动切换关键词规则引擎 |

## 五、本节小结

通过本节，学生应该了解到：
1. 这个客服系统能做什么 —— 三大功能模块的实际效果
2. 架构是什么样的 —— 前后端分离 + LLM 集成
3. 用了哪些技术 —— Spring Boot 3 + Vue 3 + LLM
4. 核心亮点是什么 —— 双引擎（LLM + 规则）、知识驱动、多轮上下文、推理可观测
5. 哪些功能是真实链路、哪些是模拟 —— Tab 标签的 tooltip 角标一目了然
