# 00 - 环境准备与运行

## 环境要求

| 工具 | 版本要求 | 说明 |
|------|----------|------|
| **Java** | JDK 17+ | 后端运行环境 |
| **Maven** | 3.8+ | 后端依赖管理 |
| **Node.js** | 20.19+ | 前端运行环境 |
| **pnpm** | 8.8+ | 前端包管理器 |
| **MySQL** | 8.0+ | 数据库 |
| **Redis** | 6.0+ | 缓存（可选，项目内置降级） |

---

## 后端启动（5步）

### 1. 创建数据库

```sql
CREATE DATABASE agent_customer_service DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

> 无需手动建表，JPA 配置了 `ddl-auto: update`，启动时自动创建 10 张业务表。

### 2. 配置本地连接

编辑 `backend/src/main/resources/application-local.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/agent_customer_service?useUnicode=true&characterEncoding=utf8&serverTimezone=GMT%2B8&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 你的MySQL密码

llm:
  enabled: true                    # 设为 true 启用大模型
  api-url: https://你的LLM网关地址/v1/chat/completions
  api-key: 你的API Key
  model: gpt-4o-mini
```

### 3. 编译项目

```bash
cd backend
mvn clean compile -DskipTests
```

### 4. 启动项目

```bash
mvn spring-boot:run
```

或在 IDE 中运行 `Application.java` 主类。

### 5. 验证后端

访问 `http://localhost:10002/api/knowledge/overview`，返回 JSON 数据即表示后端启动成功。

## 前端启动（3步）

### 1. 安装依赖

```bash
cd frontend
pnpm install
```

### 2. 启动开发服务器

```bash
pnpm dev
```

自动打开浏览器。开发环境已配置代理，`/api` 请求自动转发到 `http://localhost:10002`。

### 3. 验证前端

浏览器访问后，登录进入系统，左侧菜单可以看到「智能客服」，下有三个子菜单：

- **对话中心** → `/customer-service/chat`
- **知识中心** → `/customer-service/knowledge`
- **场景配置** → `/customer-service/scene`

## 常见问题

### Q1：后端启动报 `Communications link failure`

**原因**：MySQL 连接失败。  
**解决**：检查 MySQL 是否启动、端口是否 3306、密码是否正确。

### Q2：LLM 回复不生效，对话只返回固定文案

**原因**：`llm.enabled` 为 false 或 API Key 为空。  
**解决**：在 `application-local.yml` 中设置 `llm.enabled: true` 并填入有效的 API Key。系统有关键词兜底机制，即使 LLM 不可用也能正常对话。

### Q3：前端启动报 `pnpm: command not found`

**解决**：

```bash
npm install -g pnpm
```
